### 问题分析
- **问题1**：Schema 中联动规则可以用 `$self.value` 自引用（无 dependencies），但界面"依赖"字段为空，信息对不上——用户困惑
- **问题2**：联动规则块在属性面板占用大量空间，后续属性会越来越多

### 解决方案
**第一性原理**：联动本质 = 表达式（when 字符串）+ 效果（fulfill/otherwise），不必区分 $self 与 $deps，直接编辑 when 字符串即可。
**奥卡姆剃刀**：属性面板只显示摘要（N 条规则 + 每条一行 when→fulfill），编辑全在弹窗。

### 改动

## 第十五阶段后续修复（2026-03-27 下午）

### Bug 1：复制容器节点时子节点 x-id 未重新生成
- **现象**：复制 Card 容器后，内部子字段的 x-id 与原始容器相同，导致 overlay 查找时命中第一个容器内的节点，复制容器内的子节点"隐形"
- **根因**：`schemaUtils.duplicateNodeById` 只给顶层克隆节点赋新 x-id，`JSON.parse(JSON.stringify())` 深拷贝后子节点 x-id 保持原值
- **修复**：新增 `regenerateIds()` 递归函数，遍历克隆树所有层级重新生成 x-id
- 测试更新：`schemaUtils.test.ts` 复制 void 容器测试改为断言子节点 ID 已被重新生成

### Bug 2：内嵌节点跨容器拖拽释放后不生效
- **现象**：容器内节点拖到容器外（或根层字段旁），有拖拽视觉反馈（指示线/高亮），但释放后节点不动
- **根因**：`DesignOverlay.handleDrop` 跨层时（source.parentId !== target.parentId）：
  - 当 target.parentId === '__root__' 时，emit `reorder-nodes`，但 `sortNodesInSchema` 只在同层 properties 查找，跨层时静默返回 false
  - 当 target.parentId 是容器时，只 emit `move-to-container` 把节点移入容器末尾，但没有排序
- **修复**：
  - `schemaUtils.ts` 新增 `moveNodeAcrossContainers()`：先从原 parent 摘除，插入目标 parent properties，再调用 `sortNodesInSchema` 排序（原子操作）
  - `designerEngine.ts` 新增 `moveNodeAcrossContainers()` 方法
  - `DesignOverlay.vue` 新增 `move-across-containers` emit，跨层时统一使用此事件
  - `LowcodeDesigner.vue` 监听新事件并调用 engine 方法
