## CanvasOverlay 拖拽问题修复

### 问题
1. 容器节点无法添加子节点（drop zone 不显示）
2. 拖拽添加新容器时警告：`getSelectedStyle: 找不到节点元素`

### 根因
1. **问题1**：`props.schema` 是快照而非响应式引用，`absoluteContainers` computed 在 schema 变化后不会重新计算
2. **问题2**：`nextTick()` 只等待当前 microtask 队列，FormRenderer 可能还未渲染新节点

### 修复
- CanvasOverlay.vue:
  - schema watch：添加 `void absoluteContainers.value` 强制触发 computed 重新计算
  - selectedNodeId watch：添加 `setTimeout(0)` 确保 DOM 完全渲染后再调用 getSelectedStyle

## useDragInteraction html5-dnd drop zone 修复

### 问题
容器节点 drop zone 不显示（html5-dnd 模式下）

### 根因
html5-dnd 模式下 `handleDragEnter/Leave/Drop` 没有设置 `dragState.isDragging = true`，导致 drop zone 指示器条件 `v-if="dragState.isDragging"` 为 false

### 修复
- useDragInteraction.ts:
  - `handleDragEnter`: 设置 `dragState.isDragging = true`, `dragState.dragType = 'sort'`
  - `handleDragLeave`: 当离开 canvas 时重置 `isDragging` 和 `dragType`
  - `handleDrop`: 重置 `isDragging`, `dragType`, `sourceNodeId`

## 流式容器拖拽三区域修复（2026-04-12）

### 根因

`calcDropTarget` 对 relative void 容器只有两种判断（内部空白=拖入 / 外围=排序），
导致兄弟排序和拖入子节点互斥——鼠标在子节点上只能排序，在空白区只能拖入，无法同时支持两种操作。

旧的 `isMouseInContainerBlankArea` 函数逻辑本身没问题，但"容器边缘"无法触发排序。

### 修复

1. **useDragInteraction.ts**：
   - 删除 `isMouseInContainerBlankArea` 函数（不再需要）
   - relative void 容器改用**三区域判断**：
     - 上边缘 20%（clientY < rect.top + rect.height * 0.2）→ sort-relative before
     - 下边缘 20%（clientY > rect.top + rect.height * 0.8）→ sort-relative after
     - 中间 60% → move-into-container
   - absolute 容器逻辑不变

2. **VoidContainer.vue**：
   - 引导层 `v-if` 去掉 `!isAbsoluteMode` 限制，改为 `v-if="designMode"`
   - absolute 容器现在也会渲染引导层（由 data-drag-over 属性控制显示），与 relative 容器行为趋同
