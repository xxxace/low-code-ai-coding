# 2026-04-11 工作日志

## 修复：绝对定位容器拖拽时绿色边框不跟随 + 浮点小数

### 问题 1：拖拽绝对定位容器时绿色边框在原地不动

**根因**：`CanvasOverlay.vue` 的 `syncDraggingBox` 函数，在 nodeEl 存在时只执行一次，没有继续调度 RAF，动画循环提前终止

**修复**：`syncDraggingBox` 每次执行后无条件调度 `requestAnimationFrame`，保证拖拽全程跟随

**涉及文件**：`prototype/src/designer/CanvasOverlay.vue`

### 问题 2：positionX/Y 出现浮点小数（如 309.1999969482422）

**根因**：`useDragInteraction.ts` 的 `handleMouseUp` 中，x/y 直接用浮点运算结果 emit，未取整

**修复**：`emit('update-node-position', ...)` 时对 x/y 加 `Math.round()` 取整

**涉及文件**：`prototype/src/designer/composables/useDragInteraction.ts`

### 附带修复：CanvasOverlay.vue 重复 onUnmounted

合并两个 `onUnmounted` 为一个，并包含 raf 清理逻辑

### 问题 1 追加修复（2026-04-11 17:25）：RAF 调度位置错误

**根因**：上次修复把 RAF 放在 guard return 之后，如果 `draggingBoxRef.value` 还没 mounted 就触发 RAF，`syncDraggingBox` 直接 return 不调度，动画永远无法开始

**修复**：RAF 调度移到函数开头，保证 guard return 后动画不终止；ref 还没 mounted 时 RAF 也会持续运行直到元素出现

### 问题 2 追加修复（2026-04-11 17:25）：添加节点时浮点小数

**根因**：`useMaterialDrag.ts` 的 `handleCanvasDrop` 中，`e.clientX/Y - rect.left` 浮点运算后直接写入 schema

**修复**：对 x/y 加 `Math.round()` 取整

### 问题 1 追加修复（2026-04-11 第三轮）

**真正的三个根因**：

**A. drop zone indicator 残留**  
`handleMouseDown` 不清 `dropTarget`，上一次拖拽的 `action='move-into-container'` 残留 → 所有容器的绿色 dashed 边框一直显示  
**修复**：mousedown 时 `dropTarget.value = null`

**B. dragging box 跟随错误节点**  
`selectedNodeId` 在 `handleMouseUp` 才更新，拖拽时 `draggingBox` 使用旧节点 ID，与实际被拖拽的容器位置不同步  
**修复**：mousedown 时立即 `emit('select-node', nodeId)`

**C. absoluteContainers 包含自身**  
拖拽时 `absoluteContainers` 列表包含被拖拽的容器自身（虽然它不是 void 类型不会出现在列表，但逻辑上应该排除）  
**修复**：`absoluteContainers` computed filter 排除 `dragState.targetNodeId`
