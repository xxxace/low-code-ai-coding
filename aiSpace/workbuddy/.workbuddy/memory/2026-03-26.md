## 设计器完全空白问题修复（第四次修复）

### 问题描述

### 根本原因分析
- **script setup 顶层执行问题**：在 script setup 顶层直接调用 `loadSchema(createDemoSchema())` 可能导致执行顺序问题

`Designer.vue`:
1. 从 Vue 导入 `onMounted`

## 设计器交互失效修复（第五次修复）

### 问题描述
用户反馈"界面只能看，操作不了"—— 设计器画布内字段无法点击选中、属性面板无法交互

### 诊断过程
1. 启动开发服务器（http://localhost:5173）
2. 安装 agent-browser 测试工具，Chromium 下载失败（网络原因），改为代码分析法
3. 通读 LowcodeDesigner.vue、DesignOverlay.vue、FieldRenderer.vue、FormRenderer.vue、Designer.vue

### 根本原因分析（5个问题）

| 级别 | 问题 | 影响 |
|------|------|------|
| 🔴 致命 | FieldRenderer.formModel 类型为非可空，在设计器调用 null 时崩溃 | 阻止字段渲染 |
| 🔴 致命 | DesignOverlay.onMounted 里 refreshOverlay 时 FormRenderer 的 DOM 尚未就绪 | overlayItems 为空，叠加层无交互区域 |
| 🔴 致命 | DesignOverlay 有两个重复的 watch(schema) 声明，浪费渲染资源 | 潜在重复刷新 |
| 🟡 警告 | App.vue 和 Designer.vue 各自 provide(COMPONENT_REGISTRY_KEY)，双重注册 | 可能注册表不一致 |
| 🟡 警告 | schema 变化时 refreshOverlay 没有足够的等待时间 | 节点偶发找不到 DOM |

### 修复方案

1. **FieldRenderer.vue + VoidContainer.vue**: formModel prop 类型改为 `FormModel | null`，所有访问加 `?.` 可选链
2. **DesignOverlay.vue**: onMounted 改为 `setTimeout 50ms` 确保 FormRenderer DOM 就绪；schema watch 改为 `setTimeout 20ms`；删除重复 watch
3. **App.vue**: 保留顶层 provide，并整理 import 顺序

### 修改文件
- `prototype/src/renderer/FieldRenderer.vue`（5处修改）
- `prototype/src/renderer/VoidContainer.vue`（1处修改）
- `prototype/src/designer/DesignOverlay.vue`（2处修改）
- `prototype/src/App.vue`（import 整理）

### 验证结果
- TypeScript 类型检查：零错误 ✅
- 开发服务器：http://localhost:5173 正常运行 ✅

## 全量测试+深度修复（第六次修复，2026-03-26 16:00）

### 问题背景
用户截图显示画布字段布局挤压（字段标签乱叠在一行），确认旧 Designer.vue 仍在被使用

### 核心发现（代码完整读取后）
1. **App.vue 引用旧版 Designer.vue** — 新版 LowcodeDesigner.vue（带 DesignOverlay）从未被加载
2. **画布字段布局挤压** — canvas-renderer__preview 没有 min-width，24列 grid 在窄容器里每列只有几px
3. **DesignOverlay overflow:hidden** — 裁掉了 top:-32px 的操作按钮面板
4. **DesignOverlay 坐标计算偏差** — 用 canvasEl（canvas-container）计算偏移，但 overlay 相对 canvas-renderer 定位，存在 FormRenderer padding 偏差
5. **FormRenderer 双重 onUnmounted + 重复 watch(schema.id)** — 内存泄漏和重复初始化
6. **LowcodeDesigner 物料面板硬编码** — 只有5个组件，缺少 Textarea/Password/DatePicker 等

### 修复方案
1. **App.vue**: 改引 `LowcodeDesigner.vue`（不再用旧 Designer.vue）
2. **LowcodeDesigner.vue**: 
   - 引入 MaterialPalette 组件（动态从 ComponentRegistry 读取所有物料）
   - provide ComponentRegistry
   - canvas-renderer__preview 加 min-width: 600px
   - canvas-container 改 overflow: auto（允许滚动）
   - 移除旧的内联物料面板 CSS
3. **DesignOverlay.vue**: 
   - overflow:hidden → overflow:visible（操作按钮不再被裁剪）
   - refreshOverlay 坐标计算改为相对 overlayRef 父容器
4. **FormRenderer.vue**: 删除重复 onUnmounted 和重复 watch(schema.id)

### 修改文件
- `prototype/src/App.vue`（1处：import 改为 LowcodeDesigner）
- `prototype/src/designer/LowcodeDesigner.vue`（多处：引入 MaterialPalette，provide registry，CSS 修复）
- `prototype/src/designer/DesignOverlay.vue`（2处：overflow + 坐标计算）
- `prototype/src/renderer/FormRenderer.vue`（1处：删除重复 onUnmounted/watch）

### 验证结果
- Linter 零错误 ✅
- TypeScript 零错误 ✅
- 服务器 http://localhost:5173 在线 ✅
