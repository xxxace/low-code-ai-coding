## 设计器交互失效修复（第五次修复）

### 问题描述
用户反馈"界面只能看，操作不了"—— 设计器画布内字段无法点击选中、属性面板无法交互

### 诊断过程
1. 启动开发服务器（http://localhost:5173）
2. 安装 agent-browser 测试工具，Chromium 下载失败（网络原因），改为代码分析法
3. 通读 LowcodeDesigner.vue、DesignOverlay.vue、FieldRenderer.vue、FormRenderer.vue、Designer.vue

### 根本原因分析（5个问题）

| 级别 | 问题 | 影响 |
|------|------|------|
| 🔴 致命 | FieldRenderer.formModel 类型为非可空，在设计器调用 null 时崩溃 | 阻止字段渲染 |
| 🔴 致命 | DesignOverlay.onMounted 里 refreshOverlay 时 FormRenderer 的 DOM 尚未就绪 | overlayItems 为空，叠加层无交互区域 |
| 🔴 致命 | DesignOverlay 有两个重复的 watch(schema) 声明，浪费渲染资源 | 潜在重复刷新 |
| 🟡 警告 | App.vue 和 Designer.vue 各自 provide(COMPONENT_REGISTRY_KEY)，双重注册 | 可能注册表不一致 |
| 🟡 警告 | schema 变化时 refreshOverlay 没有足够的等待时间 | 节点偶发找不到 DOM |

### 修复方案

1. **FieldRenderer.vue + VoidContainer.vue**: formModel prop 类型改为 `FormModel | null`，所有访问加 `?.` 可选链
2. **DesignOverlay.vue**: onMounted 改为 `setTimeout 50ms` 确保 FormRenderer DOM 就绪；schema watch 改为 `setTimeout 20ms`；删除重复 watch
3. **App.vue**: 保留顶层 provide，并整理 import 顺序

## 渲染器布局深度修复（第七次修复，2026-03-26 16:11）

### 问题背景
用户截图显示设计器画布和渲染器演示的字段全部挤在一行、输入框不显示，仅显示 label 文字

### 根因分析
1. **`labelPosition: 'right'` + `labelWidth: 120px`**：el-form-item 是 flex 横排，label 占 120px，在 span 12 的 grid 单元格（约 250-350px）里 content 区域极窄（130-230px），且某些场景下 el-form 高度折叠导致 content 被裁剪
2. **el-form 没有 `width: 100%` 样式**：form 标签可能不撑满容器
3. **FlowLayout gap: 16px 16px**：列间距和行间距叠加，在窄容器里挤占空间
4. **canvas-renderer 有 overflow hidden 的父容器**：canvas-container 之前设为 overflow:auto，可能裁剪 FormRenderer 高度
5. **FieldRenderer 的 el-input 等没有强制 width: 100%**：在 el-form-item 里宽度可能不撑满

### 修复内容
1. **App.vue**：demoSchema formConfig 的 labelPosition 改为 'top'，labelWidth 改为 100
2. **FormRenderer.vue**：
   - el-form 加 `style="width: 100%"`
   - 默认 labelPosition 改为 'top'
   - `.lowcode-renderer` 加 `display: block`
   - 添加 `:deep(.el-form)` 强制 `width: 100%`
3. **FlowLayout.vue**：
   - gap 从 '16px 16px' 改为 columnGap/rowGap: '0'（间距由 field-wrapper padding 控制）
   - 加 `alignItems: 'start'`（防止 grid 行高被拉伸）
   - padding 从 16px 改为 8px 8px 0
4. **FieldRenderer.vue**：
   - 添加 `:deep(.el-input, .el-select, .el-date-editor, .el-textarea, .el-input-number)` 强制 `width: 100% !important`
   - el-form-item 加 `margin-bottom: 18px`
   - field-wrapper padding 改为 0 6px
5. **LowcodeDesigner.vue**：
   - canvas-container overflow 改回 visible（不裁剪 FormRenderer 内容）
   - canvas-renderer 去掉 min-height，改为 width: 100%
   - canvas-renderer__preview 去掉 min-width: 600px，简化为 width: 100%; display: block
   - lowcode-designer__canvas 加 min-width: 0
