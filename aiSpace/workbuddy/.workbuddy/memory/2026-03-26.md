## 第五阶段：修复布局错乱 + 自由布局不可用（2026-03-26 午）

### 问题 1：流式布局字段叠加错乱

**根因**：`LowcodeDesigner.vue` 中 `canvasRef` 绑在外层 `.lowcode-designer__canvas`，而 `DesignOverlay` 计算坐标时相对于 `canvasEl.getBoundingClientRect()`，但 overlay 本身是相对于 `canvas-renderer`（内层）定位，导致坐标系错位，所有字段的 overlay 叠加位置全错。

**修复**：将 `canvasRef` 绑定到内层 `.canvas-container`；同时重构 `canvas-renderer` 结构：让 FormRenderer 先渲染，DesignOverlay 作为后续兄弟元素叠在上面（`position:absolute; inset:0`），不再需要 `.canvas-design-overlay` 中间层。

### 问题 2：自由布局拖入字段不显示

**根因**：`handleCanvasDrop` 只设置了 `x-span: 12`，没有设置 `x-free-position`。FreeLayout 对 `x-free-position` 做了 `v-if` 过滤，所以字段全部被过滤掉不渲染。

**修复**：`handleCanvasDrop` 检测当前 `layoutMode`，在 `free` 模式下计算鼠标落点相对于 `canvas-container` 的坐标，自动注入 `x-free-position: { x, y, width: 200, height: 40 }`。

### 问题 3：FormRenderer.vue validate 错误收集为空（QA P1）

**修复**：替换 `forEach(() => {})` 空调用为 `formModel.value?.getErrors() ?? {}`。
