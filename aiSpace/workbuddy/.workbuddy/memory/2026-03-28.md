## 2026-03-28 晚间 Bug 修复（22:28）

修复两个交互 bug：

### Bug 1：流式节点（relative）design-actions 跑到画布右上角

**根因**：FieldRenderer 的 `.design-actions` CSS 用 `position: absolute; top: -32px`，但父容器没有 `position: relative`，绝对定位冒泡到 `.canvas-container`。

**修复方案（DesignOverlay 统一接管）**：
- `DesignOverlay.vue`：操作按钮条件从 `!item.isContainer && positionType === 'absolute'` 改为 `!(positionType === 'absolute' && isContainer)`，即所有流式节点（relative）和 absolute 非容器节点都显示操作按钮
- `FieldRenderer.vue`：删除 design-actions DOM 块（53-69 行）、相关 CSS 和 unused imports（Delete/CopyDocument）、废弃的 handleDuplicate/handleRemove 方法
- `VoidContainer.vue`：删除 design-actions DOM 块（el-tooltip 操作按钮 + hover 占位）、相关 CSS 和 unused imports（ElTooltip/Delete/CopyDocument）、废弃的 handleDuplicate/handleRemove 方法

### Bug 2：预览模式出现 hover 效果和操作框

**根因**：`XLayout.vue` 有两个问题：
1. `provide(DESIGN_MODE_KEY, computed(() => true))` 硬编码 `true`，无论 FormRenderer 传入什么 designMode 都被覆盖
2. `provide(SELECTED_NODE_ID_KEY, computed(() => designerEngine?.selectedNodeId.value ?? null))`：预览弹窗里的 XLayout 也 inject 到设计器 engine（因为 DESIGNER_ENGINE_KEY 由 LowcodeDesigner provide 跨层穿透），导致 selectedNodeId 不为 null

**修复**：
- `XLayout.vue`：改用 `injectDesignMode(false)` 注入真实 designMode，provide 时透传而非硬编码 `true`；selectedNodeId provide 加 `designMode.value ? ... : null` 条件
- `FormRenderer.vue`：新增 `provide(SELECTED_NODE_ID_KEY, ref(null))`，在 XLayout 上层截断穿透

154 个测试全部通过，TypeScript 零错误，5 个文件零 lint 错误。
