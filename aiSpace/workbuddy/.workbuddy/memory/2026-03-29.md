# 2026-03-29 工作日志

## 修复 absolute 容器节点 overlay 高度不同步（16:00）

### 问题描述
用户报告自由定位元素的 `absolute-overlay__item` 高度与目标节点（容器节点）不同步，只有容器节点（void-wrapper design-mode）有此问题。

### 根本原因分析
项目中存在两套定位字段：
- **旧版**：`x-free-position`（FreeLayout 架构，已废弃）
- **新版**：`x-position` + `x-position-type`（XLayout 架构，当前使用）

数据流断裂：
```
ResizeObserver 检测变化 → updateNodeFreeSize → 写入 x-free-position（旧版）
                                                     ↓
AbsoluteNodeOverlay ← 读取 x-position（新版）← 数据源不同！
```

**Bug 所在**：
1. `VoidContainer.vue:255` → `designerEngine.updateNodeFreeSize(nodeId, {...})`（写入旧版字段）
2. `FieldRenderer.vue:334` → `designerEngine.updateNodeFreeSize(nodeId, {...})`（写入旧版字段）
3. `AbsoluteNodeOverlay.vue:179` → `const pos = selectedNodeSchema.value?.['x-position']`（读取新版字段）

### 修复方案
将 `updateNodeFreeSize` 替换为 `updateNodeSize`，确保数据写入新版字段。

**修改文件**：
1. `renderer/VoidContainer.vue`（第 255 行）
   ```typescript
   // 修复前
   designerEngine.updateNodeFreeSize(nodeId, {...})

   // 修复后
   designerEngine.updateNodeSize(nodeId, {...})
   ```

2. `renderer/FieldRenderer.vue`（第 334 行）
   ```typescript
   // 修复前
   designerEngine.updateNodeFreeSize(nodeId, {...})

   // 修复后
   designerEngine.updateNodeSize(nodeId, {...})
   ```

### 验证结果
- 无 linter 错误
- 数据流已打通，容器节点的 overlay 高度与目标节点实时同步
