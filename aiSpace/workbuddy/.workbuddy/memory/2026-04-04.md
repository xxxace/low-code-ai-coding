### 拖拽排序指示器修复 - emit 事件方案（2026-04-04 下午）

**问题：** 拖拽排序时蓝色指示线始终不显示

**真正的根因（之前分析错了）：**
Vue 3 `<script setup>` 中，`defineExpose` 暴露的 ref 在父组件模板 ref 访问时是**静态解包**的，不是响应式的！

```typescript
// 父组件
const designOverlayRef = ref<{ dropIndicator: Ref<...> } | null>(null)
const dropIndicator = computed(() => {
  return designOverlayRef.value?.dropIndicator ?? null  // 只在首次赋值时执行一次！
})
```

当子组件更新 `dropIndicator.value` 时，父组件的 computed **不会重新计算**，因为它依赖的 `designOverlayRef.value` 没有变化。

**修复方案：**
1. `DesignOverlay.vue` 添加 `drop-indicator-change` 事件
2. 用 `watch(dropIndicator)` 监听变化并 emit 事件
3. `LowcodeDesigner.vue` 移除 template ref，用本地 `ref` + `@drop-indicator-change` 事件处理

**验证：** vue-tsc 零错误，vitest 149/149 通过

3. 拖拽排序的指示器不见了

**根因分析：**
两个 overlay 的职责划分和 z-index 层级存在冲突：
- `AbsoluteNodeOverlay` (z-index: 1000)：处理 absolute 节点的拖拽、缩放

1. absolute 容器本身无法被选中（因为子节点的 overlay 在上层）
2. 拖拽排序指示器被遮挡

**正确的架构方案：**
1. **保持 z-index 层级**：AbsoluteNodeOverlay (1000) 在上，DesignOverlay (900) 在下
2. **动态过滤**：当 absolute 容器被选中时，DesignOverlay **不渲染该容器内部的 relative 子节点 overlay**
3. **提高指示器 z-index**：让拖拽排序指示器显示在所有层之上

**修复内容：**

1. **DesignOverlay 添加 `isInsideSelectedAbsoluteContainer` 函数**

   - 如果是，则跳过该节点的 overlay 渲染

2. **在 `refreshOverlay` 中过滤内部节点**
   - 遍历节点时，检查是否位于选中的 absolute 容器内
   - 避免 DesignOverlay 的 overlay 拦截 absolute 容器内部的事件

3. **添加 `selectedNodeId` watcher**
   - 当选中节点变化时，重新刷新 overlay
   - 确保选中 absolute 容器时，内部子节点的 overlay 被正确移除

4. **提高 `dropIndicator` 的 z-index**
   - 从 200 提高到 9999
   - 确保拖拽排序指示器显示在所有 overlay 之上

5. **回滚 DesignOverlay 的 z-index**
   - 从 1100 恢复为 900
   - 确保 absolute 节点的交互优先
