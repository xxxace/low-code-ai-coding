> 创建时间：2026-03-26
> 目的：研究设计器画布的核心实现，为自由布局拖拽提供技术方案

## 研究背景

用户要求支持 **WinForm 风格的绝对定位布局**，用于大屏可视化、报表设计场景。
当前已调研的低代码项目（VTJ.PRO, Formily, variant-form3-vite 等）都是流式布局，
对自由布局的参考价值有限，需要补充调研。

## 技术选型问题

### 方案A: 纯 DOM + CSS 实现
- 优点：轻量、与 Vue 响应式天然集成
- 缺点：大量组件时性能可能下降
- 适用：表单场景（通常 < 50 个组件）

### 方案B: Canvas 实现（fabric.js / Konva.js）
- 优点：高性能、丰富变换能力
- 缺点：与 Vue 集成复杂、调试困难
- 适用：大屏可视化（大量图形元素）

### 方案C: 混合实现（DOM 渲染 + 第三方拖拽库）
- 优点：兼顾性能和开发效率
- 缺点：需要处理库与 Vue 的集成
- 适用：通用场景
